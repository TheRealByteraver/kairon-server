Welcome to the API.

Available endpoint(s): /tokens, /tokens/:id where id is the primary key.

Available request methods for /tokens

  • GET: get all tokens
    parameters: ?query=active (default) and ?query=inactive
    returns: an array of token objects

  • POST: add a new token. e.g. for body: { "name": "bitcoin" }
    returns: created token object on success, error object on error.
    Note: trying to create an existing but inactive token will simply reactivate that token.

Available request methods for /tokens/:id

  • GET: get a specific token, e.g. /tokens/1
    returns: a token object on success, error object on error

  • PATCH: update a specific token. e. g. /tokens/2 with body { "active": 1 }
    returns: the updated token object on success, error object on error
